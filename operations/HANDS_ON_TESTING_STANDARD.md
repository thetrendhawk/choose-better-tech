# Choose Better Tech Hands-On Testing Standard

Version: 1.0
Status: Binding
Owner: Choose Better Tech
Effective Date: 2026-07-12

## Documentation-Only Evaluation

Documentation-only evaluation may be acceptable, when clearly disclosed, for:

- architecture explanation;
- plan comparisons;
- policy comparisons;
- ownership analysis;
- audit analysis;
- conceptual guides;
- platform-support documentation;
- privacy-policy analysis.

## When Hands-On Testing Is Normally Required

Strong conclusions normally require hands-on testing for:

- VPN speed;
- VPN streaming reliability;
- latency;
- sync speed;
- conflict handling;
- offline behavior;
- app usability;
- setup friction;
- customer-support response;
- cancellation difficulty;
- import/export accuracy;
- recovery workflow;
- mobile app behavior;
- router setup;
- device compatibility;
- performance benchmarks.

## Language Restriction

When hands-on testing has not occurred, use language such as:

- documentation-based fit;
- evidence-based fit;
- current documented support;
- provisional recommendation.

Do not use `fastest`, `most reliable`, `easiest in practice`, `seamless`, `flawless`, `tested`, `proven`, or `consistently works` unless actual testing supports the claim.

## Test Record

Hands-on testing must record:

- date;
- device;
- operating system;
- app version;
- plan;
- location where relevant;
- procedure;
- result;
- failure;
- limitation;
- screenshots or logs where appropriate.

Store the record at:

`docs/testing/<slug>-test-record.md`

Site-rendering tests do not count as product testing. Testing disclosure must also comply with `operations/EDITORIAL_STANDARDS.md`.
